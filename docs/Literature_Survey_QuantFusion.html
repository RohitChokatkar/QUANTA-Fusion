Literature Survey

QUANTA-Fusion: Institutional-Grade Quantitative Finance Platform for BSE India
Rohit Chokatkar
Department of Computer Science & Engineering
April 2026
Abstract — This literature survey examines the theoretical foundations, mathematical models, and technological frameworks underlying QUANTA-Fusion, an institutional-grade quantitative finance platform for the Bombay Stock Exchange (BSE). The platform integrates ten quantitative models spanning volatility forecasting (GARCH), options pricing (Black-Scholes-Merton, Heston), market microstructure analysis (Avellaneda-Stoikov, Kyle, Glosten-Milgrom), portfolio optimization (Markowitz Mean-Variance), execution planning (Almgren-Chriss), interest rate modelling (Vasicek), and AI-driven trading signals (Reinforcement Learning with PPO). The system employs a three-language architecture (C++, Python, TypeScript) with real-time data pipelines via Fyers API, Firebase cloud infrastructure, and a React-based single-page application. This survey reviews seminal works, mathematical formulations, recent advances, and practical implementations relevant to each model, establishing the academic foundation for the platform's design decisions.
Keywords: Quantitative Finance, GARCH, Black-Scholes, Market Microstructure, Portfolio Optimization, Reinforcement Learning, BSE India, Real-Time Analytics, React, Firebase, Fyers API

1. Introduction

Quantitative finance has undergone a paradigm shift over the past three decades, evolving from simple discounted cash flow models to sophisticated algorithmic frameworks that process tick-level market data in real-time [1]. The democratization of financial data, advances in open-source scientific computing, and the rise of cloud-native architectures have made it feasible for individual researchers to build platforms that rival the analytical capabilities of institutional trading desks [2].

QUANTA-Fusion is a full-stack quantitative finance platform designed to provide institutional-grade analytics for the Bombay Stock Exchange (BSE), India's oldest and one of Asia's largest exchanges with over 5,000 listed companies and a daily turnover exceeding ₹5,000 crore [3]. The platform simultaneously executes ten quantitative models across five domains: volatility modelling, derivatives pricing, market microstructure, portfolio theory, and machine learning-based signal generation.

This literature survey is organized as follows: Section 2 reviews the system architecture and technology stack. Sections 3–7 examine the theoretical foundations of each model domain. Section 8 discusses the integration of real-time data pipelines. Section 9 covers related work in open-source quant platforms. Section 10 presents conclusions and future directions.

2. System Architecture & Technology Stack

2.2 Frontend Technologies

The user interface is built using React 18 with TypeScript, leveraging the virtual DOM reconciliation algorithm described by Abramov et al. [5]. State management utilizes Redux Toolkit following the Flux architecture pattern [6]. Interactive charting employs the Recharts library (built atop D3.js), which provides composable chart components suited for financial time-series visualization [7]. The application is deployed on Firebase Hosting, leveraging Google's global CDN for sub-100ms page loads [8].

2.3 Backend & Data Pipeline

The Python backend serves as the model engine, exposing RESTful endpoints via FastAPI [9]. Market data is ingested through the Fyers API, providing live BSE tick data, historical OHLCV candles, and real-time order book snapshots. Sentiment analysis relies on live news feeds from NewsAPI.org processed through the VADER (Valence Aware Dictionary and sEntiment Reasoner) lexicon [10].

3. Volatility Modelling

3.1 GARCH(1,1) — Generalized Autoregressive Conditional Heteroskedasticity

Engle (1982) introduced the ARCH model to capture time-varying volatility (volatility clustering) observed in financial return series [11]. Bollerslev (1986) generalized this to GARCH, which remains the most widely used volatility model in both academia and industry [12].

The GARCH(1,1) conditional variance equation is:

σ²ₜ = ω + α · ε²ₜ₋₁ + β · σ²ₜ₋₁ where: σ²ₜ = conditional variance at time t ω = long-run variance weight (ω > 0) α = ARCH parameter (sensitivity to shocks) β = GARCH parameter (persistence of volatility) ε²ₜ₋₁ = squared residual (innovation) at t-1 Stationarity constraint: α + β < 1

QUANTA-Fusion implements GARCH(1,1) using the arch Python library by Sheppard (2020) [13], fitting on 60-day rolling windows of BSE return data. The model outputs one-step-ahead volatility forecasts, annualized as a percentage. Hansen and Lunde (2005) demonstrated that GARCH(1,1) is difficult to outperform with more complex volatility specifications for daily data [14], justifying its selection.

3.2 Heston Stochastic Volatility Model

Heston (1993) introduced a closed-form stochastic volatility model that allows the variance itself to follow a mean-reverting process [15]:

dSₜ = μ · Sₜ · dt + √vₜ · Sₜ · dW¹ₜ dvₜ = κ(θ − vₜ) · dt + σᵥ · √vₜ · dW²ₜ where: Sₜ = stock price vₜ = instantaneous variance κ = mean-reversion speed θ = long-run variance σᵥ = volatility of volatility ρ = correlation between dW¹ and dW² (volatility smile/skew)

Unlike Black-Scholes, the Heston model captures the empirical volatility smile observed in options markets. Gather and Litvinova (2006) showed that introducing stochastic volatility dramatically improves calibration to BSE Nifty options [16]. QUANTA-Fusion uses Heston for generating implied volatility surfaces across strike prices and maturities.

4. Options Pricing

4.1 Black-Scholes-Merton (BSM) Model

The Black-Scholes-Merton framework [17][18], published in 1973, remains the foundation of modern options pricing theory. The model assumes geometric Brownian motion for the underlying asset, a constant risk-free rate, no dividends, and continuous hedging.

Call price: C = S₀ · N(d₁) − K · e⁻ʳᵀ · N(d₂) Put price: P = K · e⁻ʳᵀ · N(−d₂) − S₀ · N(−d₁) d₁ = [ln(S₀/K) + (r + σ²/2)T] / (σ√T) d₂ = d₁ − σ√T Greeks: Δ (Delta) = ∂C/∂S = N(d₁) Γ (Gamma) = ∂²C/∂S² = φ(d₁) / (S₀σ√T) Θ (Theta) = −S₀φ(d₁)σ/(2√T) − rKe⁻ʳᵀN(d₂) V (Vega) = S₀√T · φ(d₁) ρ (Rho) = KTe⁻ʳᵀ · N(d₂)

QUANTA-Fusion computes BSM prices and all five Greeks in real-time using SciPy's statistical functions [19]. The risk-free rate is derived from the RBI 91-day Treasury Bill rate. Hull (2021) provides the standard reference for options mathematics used in the implementation [20].

5. Market Microstructure

5.1 Avellaneda-Stoikov Market Making Model

Avellaneda and Stoikov (2008) derived optimal bid and ask quotes for a market maker seeking to maximize terminal utility while controlling inventory risk [21]:

Reservation price: r(s, q) = s − q · γ · σ² · (T − t) Optimal spread: δ(q) = γ · σ² · (T − t) + (2/γ) · ln(1 + γ/κ) where: s = mid-price, q = inventory, γ = risk aversion σ = volatility, κ = order arrival intensity T − t = time remaining, δ = bid-ask spread

This model captures the fundamental trade-off between earning the spread and managing adverse selection risk. Guéant et al. (2013) extended the model to handle multiple inventory levels and asymmetric order flow [22]. QUANTA-Fusion's implementation overlays optimal bid/ask levels on the live BSE chart.

5.2 Kyle's Lambda Model (1985)

Kyle (1985) formalized the concept of price impact and informed trading in a sequential auction model [23]. The key insight is Kyle's Lambda (λ), which measures the permanent price impact per unit of order flow imbalance:

Δpₜ = λ · (Bₜ − Sₜ) + εₜ Kyle's Lambda (λ) = σᵥ / (2σᵤ) where: Bₜ − Sₜ = net order flow imbalance (buy volume − sell volume) σᵥ = standard deviation of the fundamental value innovation σᵤ = standard deviation of uninformed noise trading

A higher λ indicates greater information asymmetry. Hasbrouck (2009) provided robust estimation methods for λ using VAR models on trade-and-quote data [24]. QUANTA-Fusion estimates λ from BSE tick data, classifying trades as buy- or sell-initiated using the Lee-Ready algorithm [25].

5.3 Glosten-Milgrom Spread Decomposition

Glosten and Milgrom (1985) modeled the bid-ask spread as a Bayesian updating mechanism, where the market maker adjusts quotes based on the probability of trading against an informed counterparty [26]:

Ask = E[V | Buy] = V_high · P(informed | buy) + V_low · (1 − P(informed | buy)) Bid = E[V | Sell] = V_high · P(informed | sell) + V_low · (1 − P(informed | sell)) Spread decomposition: Spread = Adverse Selection Component + Order Processing Component

This model establishes that the spread is compensation for adverse selection rather than just a transaction cost. Madhavan (2000) provides a comprehensive survey of market microstructure models, including extensions to the Glosten-Milgrom framework [27].

6. Portfolio Theory & Execution

6.1 Markowitz Mean-Variance Optimization

Markowitz (1952) introduced Modern Portfolio Theory (MPT), demonstrating that rational investors should select portfolios on the efficient frontier — the set of portfolios offering maximum expected return for a given level of risk [28]:

Minimize: w' · Σ · w (portfolio variance) Subject to: w' · μ = μ_target (target return) w' · 1 = 1 (full investment) wᵢ ≥ 0 for all i (no short-selling) where: w = weight vector, Σ = covariance matrix μ = expected return vector

QUANTA-Fusion solves this quadratic programming problem using SciPy's minimize function with SLSQP (Sequential Least Squares Quadratic Programming) for 2–5 BSE stocks simultaneously [19]. The efficient frontier is computed by sweeping across target returns and plotting the minimum-variance portfolio at each level. Merton (1972) extended Markowitz's framework to continuous time [29].

6.2 Almgren-Chriss Optimal Execution

Almgren and Chriss (2001) addressed the problem of executing large orders with minimal market impact, balancing execution speed against price impact cost [30]:

Minimize: E[C] + λ · Var[C] Optimal execution trajectory: nⱼ = X · sinh[κ(T − tⱼ)] / sinh(κT) where: X = total shares to execute, T = time horizon κ = √(λσ² / η) η = temporary impact coefficient σ² = volatility, λ = risk aversion

This model is the industry standard for institutional execution algorithms (VWAP, TWAP, Implementation Shortfall). Almgren (2012) extended the model to include non-linear impact functions [31]. QUANTA-Fusion generates execution schedules for large BSE block orders, showing the optimal trade trajectory and projected implementation shortfall.

7. Interest Rate Modelling & AI-Driven Signals

7.1 Vasicek Interest Rate Model

Vasicek (1977) proposed the first equilibrium model for the term structure of interest rates, incorporating mean reversion [32]:

drₜ = a(b − rₜ)dt + σ · dWₜ where: rₜ = short rate at time t a = speed of mean reversion b = long-run equilibrium rate σ = volatility of the short rate Wₜ = Wiener process Bond price: P(t,T) = A(t,T) · exp(−B(t,T) · rₜ)

While the model allows negative rates (a limitation addressed by the CIR model [33]), its analytical tractability makes it valuable for pedagogical purposes and rapid yield curve estimation. QUANTA-Fusion uses Vasicek to model RBI policy rate dynamics and estimate bond prices for BSE-listed debt instruments.

7.2 Reinforcement Learning Agent (PPO)

Schulman et al. (2017) introduced Proximal Policy Optimization (PPO), a policy gradient algorithm that provides stable training through clipped surrogate objectives [34]:

L_CLIP(θ) = E[ min( rₜ(θ) · Aₜ, clip(rₜ(θ), 1−ε, 1+ε) · Aₜ ) ] where: rₜ(θ) = π_θ(aₜ|sₜ) / π_θ_old(aₜ|sₜ) (probability ratio) Aₜ = advantage function estimate ε = clipping parameter (typically 0.2)

QUANTA-Fusion's RL agent is trained using Stable-Baselines3 (SB3) [35], with a custom Gym environment that feeds BSE OHLCV data and technical indicators as state observations. The agent outputs discrete signals: BUY, HOLD, or SELL. Fischer and Krauss (2018) demonstrated that deep reinforcement learning outperforms traditional technical analysis strategies on S&P 500 data [36], and similar results have been reported for BSE-listed stocks by Patel et al. (2020) [37].

8. Real-Time Data Pipelines & Sentiment Analysis

8.1 Market Data Integration

Real-time market data is ingested through the Fyers API v3, which provides WebSocket-based streaming for BSE equities, historical OHLCV data with 1-minute resolution, and order book depth. The platform processes data points including: open, high, low, close, volume, bid-ask spread, and total traded value. Data quality checks follow the guidelines of Brownlees and Gallo (2006) for filtering outliers in high-frequency financial data [38].

8.2 News Sentiment Analysis

Sentiment analysis is performed using the VADER (Valence Aware Dictionary and sEntiment Reasoner) lexicon, developed by Hutto and Gilbert (2014) [10]. VADER is specifically attuned to social media and news headline text, making it suitable for financial news processing without requiring deep learning model training. The compound sentiment score (ranging from −1 to +1) is aggregated across multiple news sources via NewsAPI.org and displayed alongside technical signals. Loughran and McDonald (2011) demonstrated that general-purpose sentiment lexicons require domain-specific adjustments for financial text [39], which QUANTA-Fusion addresses by weighting finance-specific terms.

10. Financial Tools & Robo-Advisory

10.1 SIP Calculator

The Systematic Investment Plan (SIP) calculator implements the standard ordinary annuity future value formula used by SEBI-regulated mutual funds in India [42]:

FV = P × [(1 + r)ⁿ − 1] / r where P = monthly investment, r = monthly interest rate (annual rate / 12 / 100), n = total months

10.2 Currency Converter

The currency converter supports 55 global currencies using cross-rate conversion through USD as the base currency, following the standard interbank convention. Exchange rates are maintained using approximate spot rates sourced from RBI and international central bank publications [43].

10.3 Robo-Advisor

The robo-advisory module implements a rule-based portfolio allocation engine inspired by Modern Portfolio Theory [28]. Users complete a 3-step questionnaire (risk tolerance, investment horizon, investment goal), and the system recommends an optimal asset allocation across six asset classes: Equity, Bonds, Gold, Cash, REITs, and International Equities. This follows the Bogle (1999) principle of strategic asset allocation [44].

11. Conclusion & Future Work

This literature survey has established the theoretical foundations underlying QUANTA-Fusion's ten quantitative models and supporting infrastructure. The platform draws from seminal works spanning six decades of quantitative finance — from Markowitz's 1952 portfolio theory through Schulman's 2017 PPO algorithm — and implements them using modern web technologies for real-time BSE analysis.

Key contributions of QUANTA-Fusion include:

(1) Simultaneous execution of 10 institutional-grade quantitative models on BSE live data via a web interface.
(2) A three-language architecture optimized for both analytical depth (Python) and user experience (TypeScript/React).
(3) Integration of market microstructure models (Avellaneda-Stoikov, Kyle, Glosten-Milgrom) typically reserved for institutional trading desks.
(4) A robo-advisory module applying Modern Portfolio Theory principles for retail investor education.

Future work includes: (a) completing the C++ speed layer for sub-millisecond tick processing, (b) implementing HMM-based regime detection for market state classification, (c) extending the RL agent to multi-asset portfolio management, (d) adding a Fama-French five-factor alpha decomposition module, and (e) incorporating real-time options chain data from BSE for live Heston model calibration.
